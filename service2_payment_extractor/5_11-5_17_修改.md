## 三、Bug 修复（实施期 + 部署期发现）

### 3.1 混签条款恢复分支误杀（5/15-5/16）

**现象**：混签合同 1 段拆 3 子节点（`定金 30%` / `进场前（首付） 50%` / `当地政府部门验收后 20%`），LLM 复核仅吐第 1 个，剩 2 个在恢复分支被错误跳过。

**根因**：I7 初版只用 `clause_text` 相似度判断"已存在"。但混签场景下 3 个子节点共用同一段 `payment_clause`，相似度恒为 1.0 → 全部误杀。

**修复**：恢复分支条件升级为 **三条件 AND**：

```python
# I7 终版逻辑
if info.clause_category != clause_category: continue
if (info.payment_type or "") != (orig_type or ""): continue   # ← 新增
if _calculate_similarity(info.payment_clause, orig_clause) >= 0.85: skip
```

`payment_type` 不同的子节点不再被误判为"LLM 改写"。

### 3.2 BOS sidecar 下载产生 ERROR 噪音（5/15）

**现象**：每次启动都打印两条 `BceServerError: The specified key does not exist`（sha256 sidecar 在 BOS 上不存在）。功能正常但日志吓人。

**修复**：`bos_helper.download_file` 增加 `quiet_on_missing=True` 参数；`_verify_pickle_sha256` 调用时传入。404 改为 debug 级日志，其它错误保持 ERROR。

### 3.3 双路径 `max_tokens` 读取不一致（5/15）

**现象**：`AppSettings.llm.max_tokens` 与 `get_llm_max_tokens()` 双路径并存，前者从 `.env` 来但无人读，后者直接读 env 但绕开 schema。

**修复**：删除 `get_llm_max_tokens()`；所有 ChatOpenAI 实例化点改读 `APP_CONFIG.llm.max_tokens` / 入参 `llm_config["max_tokens"]`。`PaymentRatioExtractor._initialize_llm` 也补上 `max_tokens` 注入。

### 3.4 死配置清理（5/16）

| 配置 | 状态 | 处理 |
|---|---|---|
| `MAX_WORKERS` | 注入但无人读 | 删除 |

| 8 | OCI 标签（image.title / image.version / image.licenses 等） |

### 6.2 部署期阻塞问题修复

#### `poetry.lock` 不一致（5/16）

**现象**：`pyproject.toml changed significantly since poetry.lock was last generated`

**修复**：Dockerfile 在 `poetry install` 前加 `poetry lock --no-update`：

```dockerfile
RUN poetry config virtualenvs.create false \
    && poetry lock --no-update \
    && poetry install --no-interaction --no-ansi --no-root --only main --no-cache
```

#### `.dockerignore` 路径无效（5/17）

**现象**：构建上下文 4.51 GB，因 `app/resources/models/bge-reranker-large/` 没被排除

**根因**：`.dockerignore` 写 `resources/models/`，Docker 只匹配上下文根，不递归

**修复**：

```
resources/models/
app/resources/models/
**/resources/models/
```

修复后构建上下文从 **4.51 GB 降到 ~30 KB**。

#### CMD JSON form（5/17）

shell form 收不到 SIGTERM 优雅停机：

```dockerfile
# 原
CMD python main.py --host 0.0.0.0 --port ${APP_PORT}
# 改为 exec form
CMD ["python", "main.py", "--host", "0.0.0.0", "--port", "8000"]
```

#### Torch GPU/CPU 版本错装（5/17）

**v9 实测**：镜像 5.86 GB，`torch.__version__ = 2.5.1+cu124`（GPU 版含 CUDA 12.4 libs ~3 GB）

**根因**：Dockerfile 预装 `pip install torch --index-url .../cpu` 抓到最新 2.12.0+cpu，但 `pyproject.toml` 约束 `torch<2.6.0`，poetry 解析时 Downgrading torch (2.12.0+cpu → 2.5.1)，主源是 GPU 版

**修复**：

```dockerfile
RUN pip install --no-cache-dir \
        "torch>=2.0.0,<2.6.0" \
        --index-url https://download.pytorch.org/whl/cpu
```

预装版本满足约束 → poetry 不再 downgrade。预期 v10 镜像 ~1.2 GB。

## 九、关键决策回顾

| 决策点 | 选择 | 理由 |
|---|---|---|
| API 契约是否变更 | **保持不变** | 用户明确约束；`payment_ratio` 仍按 % 返回 |
| pickle 安全 | sha256 sidecar 校验 | 最小侵入，不需改文件格式 |
| Semaphore 多 loop | per-loop WeakKeyDict | 兼容测试夹具与多 worker 部署 |
| ratio 内部表示 | `[0,1]` 单源 | API 边界统一 `*100` 转换 |
| prompts 改造 | 加 loader，保留 prompts.py 兜底 | 渐进迁移，不破坏现有部署 |
| Docker uid/gid | 静态 ARG（默认 1000） | 简单可靠；动态 GID 由 docker run --user 处理 |
| 模型镜像策略 | 运行时从 BOS 下载 | 镜像瘦身；建议 prod 用 PVC 共享 |
